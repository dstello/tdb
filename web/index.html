Task memory for AI agents

Your agents forget. td remembers.

A minimalist CLI that gives AI coding sessions persistent context. The next agent picks up exactly where the last one left off — no re-explaining, no lost decisions.

Get Started Read the Docs
~/project
$ td create "Add user auth" --type feature --priority P1
Created td-a1b2 · Add user auth · P1 feature

$ td start td-a1b2
Started td-a1b2 · session s-7f3c

$ td log --decision "Using JWT for stateless auth"
Logged decision on td-a1b2

$ td handoff td-a1b2 --done "OAuth flow, token storage" --remaining "Refresh rotation"
Handed off td-a1b2 · ready for next session

Why td

Built for how agents
actually work

AI sessions are ephemeral. Your project state shouldn't be.

01

Handoffs

Capture what was done, what remains, decisions made, and open questions. The next session picks up without guessing.

02

Session isolation

The implementing session cannot approve its own work — mirroring real code review and preventing unchecked changes.

03

Structured logs

Decisions, blockers, and progress are logged with intent — not buried in chat scrollback.

Get started

Two minutes to first issue

Install the CLI, create your first task, and let your agents take it from there.

Install
$ go install github.com/haplab/td@latest

$ td init
Initialized .todos/ in project root

$ td serve --port 54321
Serving on http://localhost:54321
Use
$ td create "Fix auth bug" --type bug
Created td-x8k2

$ td next
td-x8k2 · Fix auth bug · P2 bug

$ td start td-x8k2
Started · session s-2a91

Stop re-explaining.
Start handing off.
